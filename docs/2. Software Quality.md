# Software Quality

## Gates

| Gate | Command / Evidence |
|------|--------------------|
| Unit / integration tests | Project default test command (`./gradlew test`, `cargo test`, or script gate) |
| Static analysis | Gradle/Sonar, Rust fmt/clippy where applicable, or script regression |
| Contract/regression | API/event/smoke fixtures that protect external contracts |
| Evidence | `docs/evidence/` screenshots and logs |

## Test Focus

- Facade methods remain behavior-compatible after handler extraction.
- Validation chains stop on the same first error as legacy logic.
- State transitions reject invalid lifecycle movement.
- Idempotency/concurrency paths reject duplicate mutation.
- Cross-service payloads keep required fields and routing keys.

## Evidence

Evidence files live under `docs/evidence/`. File timestamps are normalized to the project evidence date convention when embedded in docs.

## Runtime Quality

Wallets, transactions, holds, payment intents, withdrawals, and provisioning events are persisted through SQLx. Idempotency uses correlation IDs/source service fields.

## Release Confidence

This module is considered ready when local tests pass and infrastructure smoke/contract gates verify the service through the gateway.
