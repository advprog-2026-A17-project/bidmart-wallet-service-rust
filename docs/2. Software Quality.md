# Software Quality

## Clean code

- Typed errors: `WalletError`, `ServiceError` in `wallet.rs` / `wallet_service.rs`.
- Money as value object in domain module.
- Router maps domain errors to HTTP status consistently.

## Secure coding

- Hold/release/convert require matching user identity headers from gateway.
- `ForbiddenAccess` on cross-user transaction reads.
- Midtrans callbacks validated in payment module (status mapping).
- Internal auction hold endpoints require gateway internal token alignment.
