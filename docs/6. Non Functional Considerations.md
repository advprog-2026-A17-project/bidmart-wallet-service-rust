# Non Functional Considerations

## ACID expectations

Wallet mutations run in SQL transactions; holds and releases must not leave negative available balance.

## Security

- Payment secrets via environment only (`MIDTRANS_*` in infrastructure `.env.example`).
- No PAN storage; Midtrans hosts card/VA flows.

## Performance

- gRPC path preferred for auction hold storm under load.
- Connection pooling via SQLx pool in `server.rs`.

## Availability

- Provisioning consumer retries on Rabbit reconnect.
- Idempotent wallet creation per user+role.

## Disaster recovery

- Postgres volume `wallet-db-data` in compose; backup strategy documented at platform level in `14. Deployment rollback.md`.
