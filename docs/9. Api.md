# API — Modul 4 (Dompet)

Base: `/api/v1/wallet` (via gateway)

OpenAPI: [openapi.yaml](./openapi.yaml)

Endpoint hold/release/convert memerlukan `X-Internal-Service-Token` (panggilan dari modul 3).

## Dompet

| Method | Path | Keterangan |
|--------|------|------------|
| GET | `/{userId}` | Saldo ringkas |
| GET | `/{userId}/detail` | Saldo + riwayat |
| POST | `/add` | Inisialisasi/adjust (admin/dev) |
| POST | `/{userId}/top-up` | Top-up langsung |
| POST | `/{userId}/top-up/intent` | Intent Midtrans |
| POST | `/{userId}/withdraw` | Penarikan |
| POST | `/{userId}/withdrawals` | Penarikan dengan rekening |

## Lelang (internal)

| Method | Path |
|--------|------|
| POST | `/hold` |
| POST | `/release` |
| POST | `/convert` |

Body hold: `userId`, `auctionId`, `bidId`, `amount`, `holdId`, `expiresAt`

## Midtrans (sandbox)

| Method | Path |
|--------|------|
| POST | `/midtrans/payments/{id}/simulate` |
| POST | `/midtrans/payments/{id}/sync` |
| POST | `/midtrans/payments/return` |

## gRPC

Port `50051` — operasi dompet untuk integrasi internal.

## Event masuk

Konsumer provisioning dari modul 1 (`WalletProvisionRequested.v1`).
